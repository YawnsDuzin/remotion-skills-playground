# 보안 정책

## 지원 버전

본 저장소는 단일 main 브랜치로 운영된다. 과거 태그는 보안 패치 대상이 아니다.

## 취약점 제보

문서·예제·스킬·스크립트에서 보안 이슈를 발견했다면 공개 이슈 대신 비공개 채널로 제보한다.

### 제보 방법

1. GitHub "Private vulnerability reporting" 사용 — `Security` 탭 → `Report a vulnerability`
2. 또는 이메일: 저장소 오너 GitHub 프로필의 연락처

### 제보에 포함할 것

- 영향 받는 파일 경로 또는 URL
- 재현 방법 (가능한 한 최소 재현 단계)
- 예상 피해 (정보 노출/권한 상승/DoS 등)
- 수정 제안 (가능 시)

## 처리 시한

- **접수 확인**: 영업일 3일 이내
- **1차 분석**: 영업일 7일 이내
- **수정 및 공개**: 심각도에 따라 1~30일

## 범위

**포함**:
- 예제 코드의 XSS/SSRF/프로토타입 오염
- 문서에서 안내하는 패턴이 보안상 위험할 때
- CI 워크플로우의 시크릿 노출
- 스킬 템플릿이 에이전트에게 위험한 코드를 만들게 유도

**제외**:
- Remotion 자체의 버그 — [remotion-dev/remotion](https://github.com/remotion-dev/remotion)으로
- 외부 서비스(Naver CLOVA, KMA API 등) 취약점 — 각 서비스 제보 채널로
- 사용자가 본 저장소 코드를 변형해 도입한 취약점

## 인정

제보자는 수정 PR 커밋 메시지 / CHANGELOG 엔트리에 (원하는 경우) 이름을 남긴다. 현금/상품 포상은 없다.
