# 기여 가이드

이 저장소는 `@yawnsduzin`이 개인적으로 유지하는 큐레이션 허브다. 외부 기여는 환영이지만 범위가 제한된다.

## 기여할 수 있는 항목

다음 라벨이 붙은 이슈만 외부 PR을 받는다.

- `good first issue` — 오타 수정, 링크 깨짐 수정
- `help wanted: korean-resource` — 한국어 자료 발굴 (블로그/영상/발표 슬라이드)
- `help wanted: screenshot` — 스크린샷/데모 영상 제공
- `help wanted: bugfix` — 예제 코드가 빌드 안 될 때

## 기여할 수 없는 항목

아래는 저자가 직접 관리하므로 PR을 받지 않는다. 제안은 이슈로 남겨달라.

- 문서의 톤/구조 변경
- 새로운 `docs/` 문서 추가
- 패턴 문서의 주관적 평가 부분

## PR 체크리스트

1. 브랜치명은 `fix/<설명>` 또는 `docs/<설명>` 형식
2. 문서 수정 시 해당 문서 상단의 "읽는 데 N분" 표기가 여전히 맞는지 확인
3. 코드 변경 시 `examples/<해당-디렉토리>`에서 `npm ci && npx tsc --noEmit`이 통과
4. 커밋 메시지는 한국어로 작성해도 무방. 시제는 현재형 (`추가한다`, `수정한다`)

## 스타일 규칙

- 문장 어미는 개조식 (`~한다`, `~된다`). 존댓말(`~습니다`) 사용하지 않음
- 번역체 표현 금지 — `~을 통해서`, `~에 대해서`, `~에 있어서`를 쓰지 말 것
- 기술 용어는 영문 병기 허용 — 컴포지션(Composition)처럼
- 각 문서는 섹션 8개 이하, 한 섹션의 코드 블록 2개 이하

문의는 이슈 또는 저장소 루트 이메일로.
