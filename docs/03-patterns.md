# 활용 패턴

> 읽는 데 약 12분

Remotion + Skills 조합은 "React로 영상을 코딩한다"는 매력 때문에 과잉 사용되기 쉽다.
이 문서는 실제로 작동하는 4가지 시나리오와, 반대로 Remotion이 오답인 케이스를 정리한다.
각 패턴은 "어떤 스킬을 조합하는가"를 매트릭스로 보여준다.

## 패턴 한눈에 보기

| 시나리오 | composition | data-fetch | tts | motion | 난이도 |
|---|---|---|---|---|---|
| 데이터 기반 일일 리포트 | ✅ | ✅ | ◻ | ◻ | 중 |
| SNS 쇼츠 자동 생성 | ✅ | ◻ | ✅ | ✅ | 상 |
| 사내 교육 영상 | ✅ | ◻ | ✅ | ◻ | 하 |
| 블로그 → 영상 요약 | ✅ | ✅ | ✅ | ◻ | 중 |

## 패턴 1: 데이터 기반 일일 리포트

매일 아침 KOSPI 종가, 환율, 주요 뉴스 헤드라인을 묶은 60초 리포트 영상을 자동 생성한다.
공공데이터 포털이나 네이버 증권 스크래핑을 `data-fetch` 스킬에 맡기고, `composition` 스킬이 받은 JSON을 차트 컴포넌트로 렌더한다.
GitHub Actions cron으로 08:00 KST 트리거 → S3 업로드 → Slack Webhook 알림까지 자동화 가능.

## 패턴 2: SNS 쇼츠 자동 생성

유튜브 Shorts / 릴스 / 틱톡용 9:16 영상을 "주제 키워드"만 주고 양산한다.
`tts` 스킬로 한국어 음성(Typecast 또는 Naver CLOVA Voice)을 깔고, `motion` 스킬이 자막 타이밍을 음성 길이에 맞게 자동 조정한다.
주의 — 음성 합성 API는 대체로 유료고 호출당 과금이다. 하루 100개 생성하면 비용이 빠르게 누적된다.

## 패턴 3: 사내 교육 영상

신입 온보딩, 보안 교육처럼 "포맷이 고정되고 내용만 주기적으로 바뀌는" 케이스가 제일 잘 맞는다.
스킬 하나(`composition`)만으로도 충분하며, 음성은 사내 녹음본을 `public/audio/`에 넣어 재사용.
수정 요청이 들어오면 Markdown 스크립트 1줄 바꾸고 재렌더 — PPT 수정보다 빠르다.

## 패턴 4: 블로그 포스트 → 영상 요약

마크다운 블로그 글을 입력받아 3~5분 요약 영상으로 변환한다.
`data-fetch`가 RSS/Markdown을 파싱하고 LLM이 핵심 문장을 뽑은 뒤 `tts`가 낭독, `composition`이 문장 단위 컷을 만든다.
"원본 블로그 링크를 CTA로 붙인다"는 규칙을 `CLAUDE.md`에 박아두면 에이전트가 매번 챙긴다.

## Remotion을 쓰지 말아야 할 때

- **인터랙티브 영상**: 시청자 클릭으로 분기되는 건 Remotion 범위 밖. `Video.js` + HLS가 낫다
- **라이브 스트리밍**: 렌더 기반이라 실시간 불가. OBS/FFmpeg 직결 파이프라인 권장
- **3D 씬 전환이 핵심인 영상**: `three.js` 가능하긴 하나 성능·디버깅 비용이 Blender 대비 크다
- **1회성 홍보 영상 1개**: After Effects 템플릿 사서 쓰는 게 시간당 단가로 유리
