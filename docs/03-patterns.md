# 활용 패턴

> 읽는 데 약 12분 · 기준일 2026-04-15

Remotion + Skills 조합은 "React로 영상을 코딩한다"는 매력 때문에 과잉 사용되기 쉽다.
이 문서는 실제로 작동하는 4가지 시나리오와, 반대로 Remotion이 오답인 케이스를 정리한다.
각 패턴은 "어떤 스킬을 조합하는가"를 매트릭스로 보여준다 — 스킬 이름은 [`skills/`](../skills) 디렉토리 실제 파일명 기준.

## 패턴 한눈에 보기

| 시나리오 | `remotion-composition` | `weather-menu` | `data-fetch` | `tts-korean` | `motion` | 난이도 | 초기 구현 |
|---|---|---|---|---|---|---|---|
| 데이터 기반 일일 리포트 | ✅ | ◻ | ✅ | ◻ | ◻ | 중 | 1~2일 |
| SNS 쇼츠 자동 생성 | ✅ | ✅ | ◻ | ✅ | ✅ | 상 | 3~5일 |
| 사내 교육 영상 | ✅ | ◻ | ◻ | ✅ | ◻ | 하 | 반나절 |
| 블로그 → 영상 요약 | ✅ | ◻ | ✅ | ✅ | ◻ | 중 | 2~3일 |

모든 스킬은 [`skills/`](../skills) 디렉토리에 실제 파일로 존재. 각 스킬 세부는 [`skills/README.md`](../skills/README.md) 참고.

## 패턴 1: 데이터 기반 일일 리포트

매일 아침 KOSPI 종가, 환율, 주요 뉴스 헤드라인을 묶은 60초 리포트 영상을 자동 생성한다.
공공데이터 포털이나 네이버 증권 스크래핑을 `data-fetch` 역할로 두고, `remotion-composition` 스킬이 받은 JSON을 차트 컴포넌트로 렌더한다.
GitHub Actions cron으로 08:00 KST 트리거 → S3 업로드 → Slack Webhook 알림까지 자동화 가능. 구현 경험상 가장 공을 들이는 구간은 "데이터 스키마 vs 레이아웃" 일치 — zod로 묶어두면 편하다.

## 패턴 2: SNS 쇼츠 자동 생성

유튜브 Shorts / 릴스 / 틱톡용 9:16 영상을 "주제 키워드"만 주고 양산한다. `examples/weather-menu`가 이 패턴의 최소 구현이다.
`tts` 역할이 한국어 음성을 깔고, `motion` 역할이 자막 타이밍을 음성 길이에 맞게 자동 조정한다.

**한국어 TTS 단가 비교 (2026-04 공개 가격표 기준, 직접 협상가 아님)**

| 서비스 | 기본 요금 | 특징 |
|---|---|---|
| Naver CLOVA Voice | 약 4원/자 (유료 플랜) | 한국어 억양 가장 자연스러움 |
| Typecast | 개인 월 19,000원부터 (월 4,000자) | 캐릭터 보이스 다양 |
| ElevenLabs 다국어 | $22/월부터 (약 10만 자) | 감정 표현 강점, 한국어 억양 살짝 어색 |

하루 100개 쇼츠(각 30초, 약 150자) 기준 한 달이면 45만 자 — 기본 플랜으로 대부분 부족하다. 실서비스로 갈 땐 엔터프라이즈 협상가가 필수.

## 패턴 3: 사내 교육 영상

신입 온보딩, 보안 교육처럼 "포맷이 고정되고 내용만 주기적으로 바뀌는" 케이스가 제일 잘 맞는다.
`remotion-composition` 하나만으로도 충분하며, 음성은 사내 녹음본을 `public/audio/`에 넣어 재사용. 수정 요청이 들어오면 Markdown 스크립트 1줄 바꾸고 재렌더 — PPT 수정보다 빠르다. 개인적으로 가장 ROI가 좋았던 패턴이다.

## 패턴 4: 블로그 포스트 → 영상 요약

마크다운 블로그 글을 입력받아 3~5분 요약 영상으로 변환한다.
`data-fetch` 역할이 RSS/Markdown을 파싱하고 LLM이 핵심 문장을 뽑은 뒤 `tts`가 낭독, `remotion-composition`이 문장 단위 컷을 만든다.
"원본 블로그 링크를 CTA로 붙인다"는 규칙을 `CLAUDE.md`에 박아두면 에이전트가 매번 챙긴다.

## Remotion을 쓰지 말아야 할 때

- **인터랙티브 영상**: 시청자 클릭으로 분기되는 건 Remotion 범위 밖. `Video.js` + HLS가 낫다
- **라이브 스트리밍**: 렌더 기반이라 실시간 불가. OBS/FFmpeg 직결 파이프라인 권장
- **3D 씬 전환이 핵심인 영상**: `three.js` 가능하긴 하나 성능·디버깅 비용이 Blender 대비 크다
- **1회성 홍보 영상 1개**: After Effects 템플릿 사서 쓰는 게 시간당 단가로 유리
- **실제 포기 사례**: 고객사 인터랙티브 튜토리얼(클릭하면 다음 챕터로)을 Remotion으로 시도했다가 상태 관리 복잡도가 폭발. 결국 Scrollytelling 라이브러리 + HTML5 `<video>`로 전환. 렌더 결과물은 Remotion이 만들고 재생만 브라우저가 하는 하이브리드도 고려 가능.
